    <article class="markdown-body entry-content" itemprop="text"><h1><a id="user-content-writing-a-script-audio-or-video" class="anchor" aria-hidden="true" href="#writing-a-script-audio-or-video"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Writing a script (audio or video)</h1>
<p>The purpose of creating a script is to create a narrative or storyline for your media. It should have a clear flow, with a distinct beginning, middle, and ending for viewers. A strong main question or theme helps to provide focus and engage viewers.</p>
<p>A standard script should have three parts:</p>
<ol>
<li>
<p>A beginning or introduction</p>
<p>a.  What is the purpose of this video, and why is it relevant</p>
<p>b.  Can start with a misconception or key question</p>
<p>c.  Can even start with an answer to highlight what the video is for</p>
</li>
<li>
<p>A middle, the main part</p>
<p>d.  What are the key points and concepts</p>
<p>e.  Make sure each part is interconnected</p>
<p>f.  Link each part to the purpose of the video</p>
</li>
<li>
<p>An end, the conclusion</p>
<p>g.  Summarise the key points made</p>
<p>h.  Relate the conclusion to the initial question, the purpose of
the video, and the wider context of the module</p>
</li>
</ol>
<h2><a id="user-content-tips-for-script-writing" class="anchor" aria-hidden="true" href="#tips-for-script-writing"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Tips for script writing</h2>
<ul>
<li>
<p>Keep it simple</p>
<ul>
<li>
<p>No jargon</p>
</li>
<li>
<p>Concentrate on simpler concepts that are easier to grasp</p>
</li>
<li>
<p>If it needs to be complicated, be careful with language</p>
</li>
<li>
<p>Make sure words are easy to pronounce</p>
</li>
<li>
<p>Remember things sound differently in writing than orally</p>
</li>
</ul>
</li>
<li>
<p>Use active language</p>
<ul>
<li>
<p>In the present tense and familiar terminologies (where possible)</p>
</li>
<li>
<p>Keep sentences short to help flow</p>
</li>
</ul>
</li>
<li>
<p>Address the audience</p>
<ul>
<li>
<p>Use terms like 'we' or 'us' or 'together'</p>
</li>
<li>
<p>Ask questions directly to the audience</p>
</li>
</ul>
</li>
<li>
<p>Length</p>
<ul>
<li>
<p>Shorter videos are more engaging</p>
</li>
<li>
<p>Keep it short -- 5/6 minutes is a good target</p>
</li>
<li>
<p>At about 130 words/min, this means a script should comprise about 700-800 words</p>
</li>
</ul>
</li>
<li>
<p>Segments</p>
<ul>
<li>Break down stories into chunks to help guide the thinking process</li>
</ul>
</li>
<li>
<p>Preparation</p>
<ul>
<li>
<p>Let your speakers know enough information to understand the video beforehand</p>
</li>
<li>
<p>For example, by sharing details about the MOOC, your module, and this document</p>
</li>
<li>
<p>Get someone unfamiliar with the topic to review your script</p>
</li>
</ul>
</li>
</ul>
<h2><a id="user-content-visual-aids" class="anchor" aria-hidden="true" href="#visual-aids"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Visual aids</h2>
<p>Think about any relevant visuals that can be used to support your text. For example, a powerpoint (or open source version) can be created with key images, bullets, and other content can be displayed alongside the audio presentation.</p>
<p>Make sure that either you own the copyright to all visual material, including photos, animations, drawings, or other videos. Otherwise, only use copyright free material, or what which you have permission to use, for you presentations.</p>
